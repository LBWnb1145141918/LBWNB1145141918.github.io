## ⚡ 快速部署

### Windows 用户（推荐）

**方法 1：使用 PowerShell 脚本（最简单）**

1. 在项目文件夹中，右键点击 `部署.ps1`
2. 选择"使用 PowerShell 运行"
3. 或者在 PowerShell 中执行：
   ```powershell
   .\部署.ps1
   ```

**方法 2：使用命令行**

```bash
# 添加所有更改
git add .

# 提交更改
git commit -m "更新内容"

# 推送到 GitHub
git push origin main
```

### 自动部署流程

执行部署脚本后，会自动：
1. ✅ 检查并添加所有更改
2. ✅ 自动提交（带时间戳）
3. ✅ 推送到 GitHub
4. ✅ 触发 GitHub Pages 自动部署

## 🔧 故障排查

### 问题 1：git push 失败

**错误信息**：`rejected main -> main (non-fast-forward)`

**解决方法**：
```bash
# 先拉取远程最新代码
git pull --rebase origin main

# 再推送
git push origin main
```

### 问题 2：部署后网站未更新

**可能原因**：浏览器缓存

**解决方法**：
1. 清除浏览器缓存（`Ctrl + Shift + Delete`）
2. 使用无痕模式访问（`Ctrl + Shift + N`）
3. 强制刷新页面（`Ctrl + F5`）

### 问题 3：GitHub Actions 失败

**检查步骤**：
1. 访问 Actions 页面查看错误日志
2. 检查是否有 `.gitmodules` 配置问题
3. 检查 `index.html` 语法是否正确

**常见错误**：
- 子模块配置错误 → 执行 `git rm -r --cached <子模块名>`
- HTML 语法错误 → 检查未闭合的标签

### 问题 4：本地修改未生效

**检查清单**：
- [ ] 代码已提交到 Git
- [ ] 已推送到 GitHub（`git push`）
- [ ] GitHub Actions 显示成功（绿色 ✓）
- [ ] 已清除浏览器缓存

## 🎯 最佳实践

### 1. 提交信息规范

使用清晰的提交信息：
```
feat: 添加新功能
fix: 修复问题
docs: 文档更新
style: 格式调整（不影响代码运行）
refactor: 重构代码
test: 添加测试
chore: 构建工具/依赖更新
```

### 2. 频繁小提交

- ✅ 每次完成一个小功能就提交
- ❌ 不要积累大量更改再提交

### 3. 推送前测试

- 本地测试无误后再推送
- 确保没有语法错误
- 确保功能正常工作

### 4. 查看部署日志

- 每次推送后检查 Actions
- 遇到失败立即查看错误日志
- 保留重要的部署记录
