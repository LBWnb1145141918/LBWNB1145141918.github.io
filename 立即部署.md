### 第 2 步：注册/登录

### 第 3 步：创建新项目

1. 登录后，点击 **"New Project"** 按钮
2. 选择 **"Deploy from GitHub repo"**

---

### 第 4 步：连接 GitHub 仓库

1. 点击 **"Configure GitHub Apps"** 或 **"Select & Install"**
2. 授权 Railway 访问您的 GitHub
3. 在仓库列表中找到：`LBWnb1145141918.github.io`
4. 点击选中这个仓库
5. 点击 **"Connect"** 或 **"Deploy"**

---

### 第 5 步：添加环境变量（重要！）

1. 在项目面板中，找到 **"Variables"** 标签页
2. 点击 **"New Variable"** 或直接添加以下变量：

**添加 3 个环境变量：**

| 变量名 | 值 |
|--------|-----|
| `SESSION_SECRET` | `scp-foundation-secret-key-2026` |
| `NODE_ENV` | `production` |
| `PORT` | `3000` |

**操作步骤：**
1. 点击 "New Variable"
2. 输入变量名：`SESSION_SECRET`
3. 输入值：`scp-foundation-secret-key-2026`
4. 点击 "Save"
5. 重复步骤添加其他变量

---

### 第 6 步：等待自动部署

1. Railway 会自动检测 `package.json` 并开始部署
2. 在 **"Deployments"** 标签页可以看到部署进度
3. 等待约 **2-5 分钟**

**部署成功的标志：**
- ✅ 状态显示为 "SUCCESS" 或 "Running"
- ✅ 出现绿色的对勾
- ✅ 显示 "Your service is live"

---

### 第 7 步：启用公网访问（生成域名）

1. 点击 **"Settings"** 标签页
2. 向下滚动找到 **"Domains"** 部分
3. 点击 **"Generate Domain"** 按钮
4. Railway 会为您分配一个免费域名，例如：
   ```
   your-project-name-production.up.railway.app
   ```

**复制这个域名！这是您网站的访问地址！**

## 🔧 常见问题解决

### Q1: Railway 检测不到 package.json？

**解决方案：**
1. 确保 `package.json` 文件存在于仓库根目录
2. 检查 `package.json` 格式是否正确
3. 在 Railway 面板中手动指定构建命令：
   - Build Command: `npm install`
   - Start Command: `node server.js`

### Q2: 部署失败怎么办？

**检查步骤：**
1. 点击 "Deployments" → 点击失败的部署 → 查看日志
2. 常见错误：
   - 缺少环境变量 → 添加 SESSION_SECRET、NODE_ENV、PORT
   - 依赖安装失败 → 检查 package.json
   - 启动失败 → 检查 server.js 是否有语法错误

### Q3: WebSocket 连接失败？

**解决方案：**
1. 前端代码已自动适配生产环境
2. 确保使用 `wss://` 协议（已自动配置）
3. 检查浏览器控制台的错误信息

### Q4: 数据库重启后数据丢失？

**原因：** Railway 使用临时文件系统

**解决方案：** 迁移到 Railway PostgreSQL

**步骤：**
1. 在 Railway 面板点击 "+ New"
2. 选择 "Database" → "PostgreSQL"
3. 等待 PostgreSQL 创建完成
4. 复制 `DATABASE_URL` 环境变量
5. 修改 `server.js` 使用 PostgreSQL（详见 DEPLOY_GUIDE.md）
