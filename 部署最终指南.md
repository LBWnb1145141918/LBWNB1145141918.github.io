## 🚀 立即部署 - 3 种方案

### 方案 1：使用 Railway 网页（最简单，推荐）

**步骤：**

1. **访问 Railway**
   ```
   https://railway.app
   ```

2. **用 GitHub 登录**
   - 点击右上角 "Login" 或 "Start a New Project"
   - 选择 "Continue with GitHub"
   - 授权 Railway

3. **创建项目**
   - 点击 "New Project"
   - 选择 "Deploy from GitHub repo"
   - 选择仓库：`LBWnb1145141918.github.io`

4. **添加环境变量**
   - 在 Variables 标签页添加：
     - `SESSION_SECRET` = `scp-foundation-secret-key-2026`
     - `NODE_ENV` = `production`
     - `PORT` = `3000`

5. **等待部署**（2-5 分钟）

6. **生成域名**
   - Settings → Domains → Generate Domain

7. **完成！** 访问分配的域名即可

### 方案 2：使用 Railway CLI（命令行）

**执行以下命令：**

```powershell
# 1. 登录（会打开浏览器）
railway login

# 2. 在浏览器中完成登录后，继续执行：

# 3. 初始化项目
railway init
# 选择 "Create new project" 或现有项目

# 4. 设置环境变量
railway variables set SESSION_SECRET=scp-foundation-secret-key-2026
railway variables set NODE_ENV=production
railway variables set PORT=3000

# 5. 部署
railway up

# 6. 分配域名
railway domain
```

## 🔧 故障排除

### 问题 1：部署失败

**解决方案：**
1. 检查环境变量是否已添加
2. 查看 Railway 部署日志
3. 确保 package.json 和 server.js 存在

### 问题 2：WebSocket 连接失败

**解决方案：**
1. 前端代码已自动适配（wss://）
2. 检查浏览器控制台的错误
3. 确保 Railway 允许 WebSocket 连接

### 问题 3：数据库重启后数据丢失

**原因：** Railway 使用临时文件系统

**解决方案：** 迁移到 Railway PostgreSQL
1. Railway 面板 → New → Database → PostgreSQL
2. 复制 DATABASE_URL
3. 修改 server.js 使用 pg 库
